Editing checklist for LaTeX papers

Checklist

These are things I have failed to do at some point in the past, and hence have learned about.

  • Have transition sentences between each section of the paper.

Math typsetting/Proofs

  • Punctuation in equations i.e.
    • periods if the equation is the last thing in the sentence.
    • commas if it isn’t.
    • commas on each line if the equations are separate between lines (particularly in proofs).
  • Explanations of each line of proof should be clear.
    • Use labels over each inequality/equality.
    • Explanation at the end of the line (with \tag*{<explanation here>}) should be well separated from the rest of the line.
  • Make up a new, intuitive symbol to replace oft-repeated math expressions.
    • For example, if my function is approximately the logarithm function, denote it as \overline{\log}.
  • Don’t start the beginning of a theorem/proof environment with an equation.
    • Good to contextualize or transition into the equation.
    • Extra whitespace if you do \begin{proof} \begin{align*} ... or something similar. - Add an extra line, perhaps with some intuition about the following equations e.g. \begin{proof}

Dealing with an appendix

  • Every appendix section should be referenced in the main body of the paper.
  • There should be no broken references. This includes both internal references to labels, and bibliographic references.
    • Ctrl + F for ?? usually finds them.
    • Especially on Overleaf, have both main paper and appendix in one document, and split later with pdftk. xcite works inconsistently on Overleaf.

Conference vs. arXiv version

Sometimes we want the arXiv version to be agnostic of the conference format. I’ve been told it’s good practice to do this, although many people don’t and it doesn’t seem to be particularly detrimental to those that don’t. However, if you want to maintain separate versions, this is how I accomplish it.

I store all content of the paper in a tex/ directory. Hence, my folder for my paper looks something this:

- tex/
|--- main.tex
- common.sty
- arxiv_draft.tex
- conference_draft.tex

Then, I usually put the following snippet at the top of my common.sty style file:

\usepackage{etoolbox}
\newcommand{\ifarxiv}[2]{
    \ifdef{\arxivversion}{#1}{#2}
}

% Use the first TeX statement for conference version and the second TeX statement for the arXiv version as to handle formatting differences.

Thus, the \ifarxiv command becomes the first argument if \arxivversion when the first statement is declared, and otherwise becomes the second statement.

My arxiv_draft.tex would then include

\usepackage{common}
\newcommand\arxivversion

On the other hand, conference_draft.tex only includes \usepackage{common}. Both *_draft.tex files would need to have \input{tex/main.tex} somewhere in the body of the file. Now I can use \ifarxiv{<arxiv version of content>}{<conference version of content>} inside of tex/main.tex to have one content file for two different versions.
